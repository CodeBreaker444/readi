```
  Assessment    Flight plan   Execution    Asset care

```

---

## Phase 1 — Evaluation

Assess whether a proposed operation is safe and compliant before flight planning begins.

- `evaluation_status`: `NEW → PROCESSING → DONE`
- Each evaluation contains tasks (`evaluation_action`) with checklist results and evidence in `dependencies` JSON
- Status = `DONE` unlocks the planning phase

---

## Phase 2 — Planning

Build the detailed flight plan.

| Status | Meaning |
|---|---|
| `NEW` | Draft just created |
| `PROCESSING` | Under internal review |
| `REQ_FEEDBACK` | Awaiting sign-off |
| `POSITIVE_RESULT` | Approved — ready for mission creation |
| `NEGATIVE_RESULT` | Rejected — must be revised |

## Phase 4 — Maintenance

| Type | Triggered by |
|---|---|
| `preventive` | Calendar schedule |
| `corrective` | Damage discovered during/after a mission |
| `inspection` | Regulatory periodic check |
| `calibration` | Sensor/compass recalibration |

**Component cycle:**
```
Mission completes
  → usage_hours updated on components
  → if usage_hours >= maintenance_cycle_hour: alert sent
  → MM creates maintenance_ticket → system = MAINTENANCE
  → ticket closed → system = OPERATIONAL
```

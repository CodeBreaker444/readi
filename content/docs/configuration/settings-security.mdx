## Main capabilities

- Password policy and change flows
- Multi-factor authentication (MFA) setup and verification
- API key management for third-party integrations
- **Authorization PIN setup** — required before any mission status change can be performed

## Authorization PIN

Every user who performs mission status changes must configure a 6-digit Authorization PIN.

If a user tries to move a mission without a PIN configured:
> "Authorization PIN not set up"

Go to **Settings → Security** to configure it.

## Primary app routes

- `/settings/security`
- `/auth/setup-2fa`
- `/auth/change-password`

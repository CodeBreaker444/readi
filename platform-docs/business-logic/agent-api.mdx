The AI Agent answers natural language questions about missions, maintenance, compliance, and safety — scoped to the user's role and organisation.

## Pipeline

```
User question
    │
[1] CLASSIFY INTENT
    DATABASE | PROCEDURE | WEB_SEARCH | DECOMMISSIONED | OTHER
    │
[2] SELECT TABLE(S)
    LLM picks 1–2 tables from the role's allowed list
    │
[3] GENERATE QUERY PLAN (per table)
    { table, aggregation: COUNT|LIST, date_filter, extra_filter, row_limit }
    │
[4] EXECUTE (Supabase PostgREST)
    - fk_owner_id always applied
    - PIC: pilot_mission auto-filtered to own missions only
    - Blocked columns stripped
    - Max 15 rows (50 hard cap), data capped at 8,000 chars
    │
[5] SYNTHESISE ANSWER
    Groq LLM with DB data + company procedure docs + optional web search
    + decommissioned systems data (if DECOMMISSIONED intent detected)
```

## Coverage Gaps

| Missing Data | Impact |
|---|---|
| `evaluation` / `evaluation_action` | Cannot answer "what evaluations are open?" |
| `user_qualification` | Cannot answer "which pilots have expired qualifications?" |
| `pilot_declaration` (non-PIC roles) | OPM cannot ask "did pilot X file a pre-flight declaration?" |
| `planning_test_logbook` | Test flight details unavailable |
| `incident_flag`, `rth_unplanned`, `link_loss`, `deviation_flag` | Not in TABLE_SCHEMA — LLM planner doesn't know these exist |
| `battery_charge_start/end`, `fuel_used` | Missing from TABLE_SCHEMA |
| Cross-table JOINs | Agent queries 1–2 tables sequentially — three-table questions may be incomplete |

**Blocked columns (never returned):**
`password_hash`, `auth_user_id`, `key_`, `api_key`, `session_token`, `purchase_price`, `actual_cost`, `estimated_cost`

## Built-in Compliance Audit Rules

The synthesiser automatically applies these checks:

1. **Wind/weather compliance** — mission flew while a `WIND_ALERT` was `OPEN` → `VIOLATION:`
2. **Altitude limit** — `max_altitude > 120 m` → `VIOLATION:`
3. **Maintenance gate** — drone had an open `HIGH` priority ticket during flight → `VIOLATION:`
4. **Decommissioned systems** — queries `tool_component` for `expiration_date <= today` and groups expired components by system

The AI Agent answers natural language questions about missions, maintenance, compliance, and safety — scoped to the user's role and organisation.

## Pipeline

```
User question
    │
[1] CLASSIFY INTENT
    DATABASE | PROCEDURE | WEB_SEARCH | DECOMMISSIONED | OTHER
    │
[2] SELECT TABLE(S) — LLM picks 1-2 from role allowed list
    │
[3] GENERATE QUERY PLAN
    { table, aggregation: COUNT|LIST, date_filter, extra_filter, row_limit }
    │
[4] EXECUTE (Supabase)
    - fk_owner_id always applied
    - PIC: pilot_mission filtered to own missions only
    - Blocked columns stripped
    - Max 15 rows, data capped at 8,000 chars
    │
[5] SYNTHESISE (Groq llama-3.3-70b)
    DB data + company procedures + optional web search + decommissioned data
```

|---|---|
| **PIC** | `pilot_mission` (own only), `pilot_mission_result`, `planning`, `pilot_declaration`, `notification`, `repository_file` |
| **OPM** | All of the above + `calendar_shift`, `checklist`, `luc_document`, `users`, `tool`, `tool_maintenance`, `tool_component`, `maintenance_ticket`, `audit`, `compliance_requirement`, `safety_report`, `alert_log`, `training`, `client` |

| **AM** | `pilot_mission`, `planning`, `tool`, `tool_maintenance`, `luc_document`, `audit`, `compliance_requirement` |
| **CMM** | `audit`, `audit_finding`, `compliance_requirement`, `compliance_evidence`, `compliance_status_log`, `spi_kpi` |

## Coverage Gaps

| Missing Data | Impact |
|---|---|
| `evaluation` / `evaluation_action` | Cannot answer "what evaluations are open?" |
| `user_qualification` | Cannot answer "which pilots have expired qualifications?" |
| `pilot_declaration` (non-PIC) | OPM cannot ask "did pilot X file a pre-flight declaration?" |
| `incident_flag`, `rth_unplanned`, `link_loss`, `deviation_flag` | Not in TABLE_SCHEMA — planner unaware |
| `battery_charge_start/end`, `fuel_used` | Missing from TABLE_SCHEMA |

**Blocked columns:** `password_hash`, `auth_user_id`, `key_`, `api_key`, `session_token`, `purchase_price`

## Built-in Audit Rules

1. **Wind compliance** — mission flew while `WIND_ALERT` was `OPEN` → `VIOLATION:`
2. **Altitude limit** — `max_altitude > 120 m` → `VIOLATION:`
3. **Maintenance gate** — drone had open `HIGH` priority ticket during flight → `VIOLATION:`
4. **Decommissioned** — queries `tool_component` for `expiration_date <= today`, groups by system

---
title: "Checklist Logic"
description: "How checklists are created, assigned, and completed."
---

Checklists are reusable templates attached to missions, evaluations, or standalone tasks.

## Creation

1. OPM or Admin creates a checklist in **Organisation Settings → Checklists**
2. Fields: `checklist_code`, `checklist_desc`, `checklist_json` (items/sections), `checklist_ver`, `checklist_active`
3. Editing creates a new version — not an overwrite

## Assignment

| Type | DB field |
|---|---|
| To a mission plan | `checklist.fk_planning_id` |
| To a pilot | `checklist.fk_user_id` |
| To an evaluation | `evaluation_action.dependencies` JSON |

## Completion

Results written into `evaluation_action.dependencies`:

```json
{
  "checklist_result": { "item_1": true, "item_2": true },
  "checklist_result_at": "2024-01-15T10:30:00Z"
}
```

For pre/post-flight checklists, `pre_flight_check_ok` / `post_flight_check_ok` on the mission record is updated.

A **Mission** is a single flight event tied to a pilot, a drone, and a flight plan (`pilot_mission`).

## Mission Fields

| Field | DB Column | Meaning |
|---|---|---|
| Mission ID | `pilot_mission_id` | Auto-incremented PK |
| Mission code | `mission_code` | Human-readable unique code |
| Linked plan | `fk_planning_id` | The flight plan this executes |
| Pilot | `fk_pilot_user_id` | Assigned PIC |
| Drone | `fk_tool_id` | Which system was used |
| Planned start | `scheduled_start` | Target takeoff datetime |
| Actual start | `actual_start` | Real takeoff timestamp |
| Actual end | `actual_end` | Real landing timestamp |
| Flight time | `flight_duration` | Minutes in the air |
| Distance | `distance_flown` | Kilometres flown |
| Max altitude | `max_altitude` | Peak altitude in metres (>120 m = compliance flag) |
| Pre-flight check | `pre_flight_check_ok` | Checklist signed off before flight |
| Post-flight check | `post_flight_check_ok` | Checklist signed off after flight |
| Incident flag | `incident_flag` | Safety incident occurred |
| RTH unplanned | `rth_unplanned` | Drone returned home unexpectedly |
| Link loss | `link_loss` | Comms link was lost |
| Deviation | `deviation_flag` | Flight deviated from planned route |
| Battery | `battery_charge_start` / `battery_charge_end` | Battery % at takeoff and landing |

## Mission Statuses

Organisation-defined in `pilot_mission_status`.

| Status | `is_final_status` | Meaning |
|---|---|---|
| `PLANNED` | false | Scheduled, not yet started |
| `IN_PROGRESS` | false | Drone is airborne |
| `COMPLETED` | true | Flight finished successfully |
| `CANCELLED` | true | Called off before takeoff |
| `FAILED` | true | Started but did not complete normally |

`is_final_status = true` marks terminal states — no further transitions expected.

## Step-by-Step Lifecycle

```
1. CREATION
   Mission linked to planning_id, status = PLANNED, pilot + drone assigned

2. PRE-FLIGHT
   pre_flight_check_ok = true, pilot_declaration (Pre-flight) filed
   System must be OPERATIONAL

3. TAKEOFF → IN_PROGRESS
   Actual start time recorded, Authorization PIN required
   FlytBase telemetry begins, battery charge at takeoff recorded

4. LANDING → COMPLETED
   Actual end time recorded, Authorization PIN required
   flight_duration, distance_flown, max_altitude recorded

5. POST-FLIGHT
   post_flight_check_ok = true, pilot_declaration (Post-flight) filed
   Results stored in pilot_mission_result

6. FAILURE
   status = FAILED, incident flags set, safety report may be filed

7. CANCELLATION
   status = CANCELLED, actual_start remains NULL
```

## Mission Results (`pilot_mission_result`)

| Field | Meaning |
|---|---|
| `result_type` | Category (imagery, thermal, RGB, survey) |
| `quality_score` | 0–100 rating |
| `data_size` | Bytes of raw data collected |
| `processing_status` | `Raw → Processing → Ready` |

Pilots (role `PIC`) are `public_users` records. Readiness is determined by qualifications, training, and declarations.

## Qualifications (`user_qualification`)

| Field | Meaning |
|---|---|
| `qualification_name` | Name of the cert/licence (A2 CofC, GVC, BVLOS) |
| `expiry_date` | When it lapses |
| `status` | `Active` / `Expired` / `Suspended` |

A pilot is **qualified** when all required qualifications have `status = Active` and `expiry_date > today`.

## Training (`training_attendance`)

| Field | Meaning |
|---|---|
| `attendance_status` | `present` / `absent` / `excused` |
| `completion_status` | `completed` / `in_progress` / `failed` |
| `score` | 0–100 |
| `certification_issued` | Whether this session generated a cert |
| `certification_expiry` | When that cert expires |

---

## Pilot Declarations

Filed before and after every flight via `pilot_declaration`.

| `declaration_type` | When filed |
|---|---|
| `Pre-flight` | Before takeoff — confirms fitness, weather, system checks |
| `Post-flight` | After landing — confirms completion, anomalies noted |
| `Medical` | Periodic fitness declaration |

<Warning>
If a pilot has not filed a daily declaration, mission status changes on the Operations Board are blocked.
</Warning>

## Pilot States in a Mission

```
ASSIGNED → PRE-FLIGHT → AIRBORNE → POST-FLIGHT → DONE
    │           │            │           │
  mission    pilot_decl   in progress pilot_decl
  created    (Pre-flight)             (Post-flight)
```

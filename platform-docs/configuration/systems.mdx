## Main capabilities

- System and component inventory management
- Component expiry tracking — expired components push a system to `NOT_OPERATIONAL`
- Maintenance dashboard and ticket lifecycle: `open → in_progress → resolved → closed`
- System map for location visualisation

## Key logic

See [System Status Rules](/platform-rules/system-status-rules) for how `OPERATIONAL / MAINTENANCE / NOT_OPERATIONAL` is computed.

## Primary app routes

- `/systems/manage`
- `/systems/map`
- `/systems/maintenance-dashboard`
- `/systems/maintenance-tickets`

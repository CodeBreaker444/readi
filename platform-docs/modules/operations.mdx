Operations is where missions move from planned to completed. Three views of the same data.

## Views

| View | Purpose |
|---|---|
| **Operations Table** | Structured list of all missions with filters |
| **Daily Board** | Kanban drag-and-drop — move missions between status columns |
| **Calendar** | Timeline view for scheduling and crew visibility |

## Mission status changes

Changing a mission status on the Daily Board requires an **Authorization PIN**. See [Authorization Code](/platform-rules/authorization-code) for the full flow.

**Blocked cases:**
- System is `NOT_OPERATIONAL` or `MAINTENANCE`
- Pilot has not filed a daily declaration
- User has no Authorization PIN configured

## Primary app routes

- `/operations`
- `/operations/board`

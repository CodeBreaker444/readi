## 🚀 What's New

### Release Logs Page
- New full-screen two-panel changelog page at `/releases`
- Left panel: searchable, filterable version list with radio-dot selection
- Right panel: release detail with highlight cards, tabbed sections (What's New, Improvements, Bug Fixes, Details), Download MD button, and helpful vote
- Markdown files in `releases/` folder auto-parsed — add a new `.md` to publish a release
- Accessible to both SUPERADMIN and ADMIN roles
- Visible in sidebar under Configuration after Settings

### Notification Toasts — Grouped by Category
- New `GroupedNotificationToast` component replaces individual per-notification toasts
- Multiple simultaneous notifications batch into one card per category (e.g. "5 Maintenance alerts")
- Toast cards match the app theme — white/dark card with coloured left-border accent
- Clicking a toast navigates directly to `/notifications?tab=<category>`

### Notifications Page — Tabbed Categories
- New category tab bar: All, Maintenance, Mission, General, Assignment, Other
- Tabs auto-hide if a category has no notifications
- Unread count badge per tab (red when unread, grey when all read)
- URL param sync — `/notifications?tab=maintenance` pre-selects the correct tab
- Clicking a grouped toast or dropdown category card opens the right tab directly

## 📈 Improvements

- Notification dropdown now shows the last 5 individual notifications with category badges, unread dot indicators, and inline mark-as-read buttons
- Notification dropdown loading replaced with shadcn Skeleton rows matching the item layout
- Notification dropdown footer shows unread count inline ("View all notifications — 3 unread")
- AI agent now correctly distinguishes COUNT vs LIST queries based on question intent
- AI agent date range detection improved — supports today, this week, this month, this year, last month
- AI agent ordering support added — handles "latest", "most recent", "first" queries with correct sort direction
- AI agent row limit capped at 15 by default for list queries; COUNT queries are unrestricted
- Increased table picker token limit for more accurate table selection
- Post-mission service updated to correctly store KPI values after flight completion

## 🐛 Bug Fixes

- Fixed Flytbase flights not listing in Control Center — now falls back to company-level credentials when user-level credentials are not set
- Fixed maintenance dashboard not loading correctly
- Fixed flight logs not showing in the operations board
- Fixed flight logbook missing records when filtering by client or mission plan
- Fixed flight request logs API build error

## 🔐 Security

- Prisma ORM integrated for additional database access layer

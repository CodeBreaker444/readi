## 🚀 What's New

### AI Agent — Markdown Responses
- Agent replies now render full GFM markdown — bullet lists, numbered lists, headings, bold, italic, inline code, code blocks, tables, blockquotes, and links
- Dark and light theme styles applied correctly per bubble via `.chat-md-dark` and `.chat-md-light`
- User messages remain plain pre-wrapped text, unaffected

### Component — Expiration Date & Auto-Decommission
- Components now have an optional Expiration Date field in both Add and Edit modals
- On save: if the selected date is today or earlier the component status is immediately set to DECOMMISSIONED
- On fetch: expiration is re-evaluated on every load — status is always correct even if the component was never edited after expiry
- Decommissioned components are already excluded from parent-component selectors and system-availability checks

### Maintenance Ticket — Basic vs Routine Filtering
- Basic ticket (31-day) component picker now only shows components with no maintenance cycle or a 31-day day-based cycle
- Components on any other interval (e.g. 90 days, 365 days, hours-only, flights-only) are excluded and remain tied to their system activation date
- Standard and Extraordinary tickets are unaffected — all active components remain visible
- Ticket close logic respects the same filter — counters of non-31-day components are not reset when a Basic ticket is closed
- Changing ticket type in the New Ticket modal immediately re-fetches the filtered component list

### i18n — Team & Settings Pages
- Language switch (EN / IT / DE) now covers: User Management, Client Management, API Key Manager, DCC Integration Settings, Pin Setup, Security Settings, Integrations Settings, and Client/User table columns
- Over 180 new translation keys added across `en.json`, `it.json`, and `de.json`

## 📈 Improvements

- Batch mission creation capped at 10 missions per request, consistent with the batch delete limit
- Company name in the top bar is now responsive on small screens — truncated with a max-width constraint to prevent overlap with sidebar and action buttons
- Maintenance table skeleton now respects dark/light theme — pulse bars and borders match the active theme

## 🐛 Bug Fixes

### Maintenance Dashboard — Systems Not Listing
- Fixed an issue where the maintenance dashboard showed no systems for companies with multiple clients assigned
- The dashboard now correctly shows all active systems belonging to the company
- Decommissioned components are now excluded from the maintenance dashboard
- Detached components are now excluded from the maintenance dashboard
- Legacy components with encoding issues are now parsed correctly
- When creating a ticket for multiple components, all components now receive their own activity log entry — previously only the first component in a batch was recorded

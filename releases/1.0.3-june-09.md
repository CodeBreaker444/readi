## What's New

### AI Agent — Decommissioned Component & System Awareness

The AI agent can now answer questions about decommissioned systems and expired components directly.

- A new `DECOMMISSIONED` intent is detected when questions reference decommissioned, expired, non-operational, or out-of-service components or systems.
- The agent queries `tool_component` for all components whose `expiration_date` is on or before today's date.
- Results are grouped by system (tool): each expired component is listed with its name, type, serial number, expiration date, and the parent system it belongs to.
- Any system with at least one expired component is treated as non-operational/decommissioned in the response.
- If no expired components exist, the agent explicitly states "No decommissioned systems or components found."

### Operation Logbook — System-Based Filtering with Real Status

- The system (drone/vehicle) filter in the Operation Logbook now returns the actual operational status of each system from the `tool_status` table, replacing the previously hardcoded `"OPERATIONAL"` value.
- Systems marked as `NOT_OPERATIONAL` in the platform will now appear with their correct status in the logbook filter, allowing accurate filtering by system state.

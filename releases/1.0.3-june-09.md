## What's New

### AI Agent — Decommissioned System Detection

The AI agent now recognises a new intent: `DECOMMISSIONED`. When a user asks questions like:
- "Which systems are decommissioned?"
- "Are there any expired components in my fleet and which systems are affected?"

The agent queries `tool_component` directly for components where `expiration_date <= today`, groups results by system, and returns a structured list of decommissioned systems with their expired components and expiry dates.

A system is considered decommissioned when at least one active component (`component_active = 'Y'`) has an `expiration_date` on or before today.

### Operation Logbook — Real Drone Status

The drone status shown in the Operation Logbook filters now reflects the actual value from `tool_status.status_code` instead of a hardcoded `'OPERATIONAL'` fallback.

## Bug Fixes

### Activation Email Errors Now Surfaced

When creating a new user, email delivery failures were previously caught silently and discarded. The API would return `send_mail_to_new_user: true` even when the email never sent.

Fix:
- `sendUserActivationEmail` now throws on Resend API errors with the full error detail logged
- `createUser` catches email failures gracefully — the user account is still created, but the response now includes `emailSent: false` and `emailError: "..."` with the exact Resend error
- The user creation API route returns `email_error` in the response body so the admin UI can show a warning

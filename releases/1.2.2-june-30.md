## What's New

### Centralized Control Center Configuration (C2 Config)

We have overhauled how fleet tracking credentials are managed by shifting connection setups away from individual user profiles into a dedicated administrative panel.

- **Admin-Managed Organizations** — Administrators can now securely add and manage multiple independent fleet tracking organizations by saving custom names, unique identifiers, and connection keys in one secure dashboard.
- **Granular Access Permissions** — A new access controls tab enables administrators to link specific organizations to individual team members. Once assigned, users instantly gain visibility into their authorized operational tracking views.
- **Side-by-Side Filtering** — Inside the flight logs view, a streamlined organization sidebar allows operators to quickly toggle between authorized networks to dynamically retrieve and view the correct logs.
- **Decluttered Profiles** — Individual user settings have been completely simplified by removing individual connection options, creating a single source of truth across the workspace.

---

### Direct Mission Linking from Recent Flights

We have introduced a fast, automated workflow to match completed flight logs directly back into your core mission registry.

- **One-Click Linking** — A new entry button positioned next to the refresh panel opens a window showcasing completed missions that are currently missing a matching flight log.
- **Strict Matching Rules** — To eliminate data confusion, the system automatically checks the drone identifiers, showing only the records that share the exact matching hardware profile as the recorded log.
- **On-the-Spot Adjustments** — Within this window, operators can seamlessly choose an ongoing record or spin up a new mission on the spot using the standard entry interface.
- **Automatic Audit Alerts** — Standard operational missions link together smoothly. For advanced operations, links established retroactively after a flight has already finished will automatically register a compliance notification within the historical record for maximum oversight.
- **Unified Syncing** — Once linked here, the connection instantly synchronizes everywhere, automatically filling out information across all historical mission logs.

---

### D-Flight Integration for Regulatory Compliance

Fully automated mapping integration with official international frameworks is now live. This guarantees seamless compliance with standard drone fleet mapping and reporting regulations for complex flights.

---

### Migration to Improved Documentation

We have completely transitioned our technical guides and help manuals to a brand-new internal layout designed to give your team significantly faster load times and highly precise keyword search results.

## Improvements

### Streamlined Dashboard & Navigation Hierarchy

- **Smarter Submenus** — Relocated the main asset tracking view to rest directly underneath the recent activities directory as a clear submenu option, keeping navigation paths intuitive.

---

### Activity History Logs

- **Asset Relocation Insights** — Moving parts or equipment into an inventory warehouse now correctly generates an immediate, permanent confirmation record inside the system audit log.
- **System Position Tracking Updates** — Updated system log format to capture explicit movement data: `Updated system 'DJI Matrice 4TD-copy' — position: 37.497861, 15.065002, status: OPERATIONAL`. The system audit framework now retains the historical coordinate profile, allowing operators to view the previous position directly within the details modal for comprehensive route tracking.

---

### Platform Stability Safeguards

- **Faster Action Responses** — Greatly improved response times for managing tracking lists or removing items and layouts by reworking how information updates simultaneously behind the scenes.
- **Data Validation Guardrails** — Added automated input verifications before onboarding new tracking organizations or initiating outbound requests, preventing layout lockups and displaying explicit, clear guidelines if data fields are missing.

---

## Bug Fixes

### Hardware Validation for Records

We applied strict background validation checks to all manual log uploads and network streams. The system will automatically decline any log profile where the internal hardware identifiers do not line up with the physical system selected for that mission.

---

### Timezone-Correct Schedules

Fixed a display bug where upcoming calendars on the main view fell back to default regional clocks rather than reflecting the operator's current local timezone.

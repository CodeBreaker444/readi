## What's New

### D-Flight Fleet Now Shows Your Full Registry — with One-Click Import

The D-Flight Fleet page no longer only shows drones you've already added to Readi — it now lists every active drone registered in your D-Flight account, and clearly marks whether each one is already linked to a Readi system by serial number, showing its DRC once matched. For any drone found in D-Flight but missing from Readi, a new **Import** button opens a pre-filled wizard that creates the System and Drone (Aircraft) Component in one step, automatically matching (or letting you create) the right Brand/Model along the way.

### Track Insurance Details for Every Component

Components can now carry their own insurance information — insurer name, company, and expiry date — right alongside their other details. This is completely optional and lives in a tidy, collapsible "Insurance" section on the Add and Edit Component screens, so it stays out of the way until you need it.

### Maintenance Dates Now Visible on the Dashboard

The dashboard now shows the upcoming Basic, Standard, and Extraordinary maintenance dates for your systems at a glance, so you can see what's coming up without opening each system individually.

### Clearer Labeling for Manually Logged Flights

In Recent Flights, flight logs that were added by hand are now clearly tagged as "Manual Mission", so it's easy to tell at a glance how a flight entry made it onto the list.

## Improvements & Fixes

### Authorization Code Now Required to Attach a Log to an Existing Mission

To help prevent a flight log from being attached to the wrong mission by mistake, an authorization code is now required whenever you attach a log to a mission that already exists. This adds a simple safety check before the log and mission are linked together.

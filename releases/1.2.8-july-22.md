## What's New

### System Email Notification Configuration

You can now configure email notifications for system-related events such as maintenance ticket creation, intervention start/end, and password changes. Control which events trigger emails and who receives them based on roles or specific user IDs.

### Operations Email Notification Configuration

Email notifications for operations events are now fully configurable. Enable or disable emails for mission creation, mission start, mission completion, calendar events, and more. Set recipients by role or specify individual users to notify.

### Module-Level Feature Flags

Two new feature flags have been added to control email notifications at the module level:
- `operation_email_enabled`: Enable or disable all operations-related email notifications
- `system_email_enabled`: Enable or disable all system-related email notifications

These flags provide a master switch for each module, allowing you to quickly enable or disable email notifications without changing individual event configurations.

### Granular Event Control

For each module, you can control which specific events trigger email notifications. For operations, this includes events like `mission_created`, `mission_started`, `mission_completed`, and `calendar_event_created`. For system events, this includes `maintenance_ticket_created`, `intervention_started`, `intervention_ended`, and more.

### Role-Based and User-Specific Recipients

Configure who receives email notifications by selecting roles (e.g., Admin, Manager, Technician) or by specifying individual user IDs. This gives you flexibility to ensure the right people are notified for each type of event.

## Improvements

- Email sending logic has been integrated into the main operation creation and status update flows
- Mission creation now triggers mission created emails when configured
- Mission status changes (start/complete) now trigger appropriate email notifications
- Calendar event creation triggers calendar event created emails
- All_email sending includes error handling to prevent failures from blocking main operations
